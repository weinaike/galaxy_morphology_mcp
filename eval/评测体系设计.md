# 评测体系设计文档

## 概述

评测体系分为三个层次，加一个独立的 RL Reward 模块：

1. **离线评测（步骤级，不跑 GALFIT）** — 快速衡量模型输出与 GT 的文本匹配度
2. **执行评测（步骤级 teacher-forcing，跑 GALFIT）** — 类比 UI-S1 的 AndroidControl-High（每步用 GT 上下文，报 SR）。**当前优先实现**
3. **SOP 半在线轨迹评测（模型自主历史）** — 类比 UI-S1 的 SOP (Sequential Online Performance)，从 GT root 出发，模型用自己的历史 rollout，遇错即停，报 PG/TSR/Score。**暂不实现**
4. **RL Reward 设计与对齐验证** — 用 rule-based 指标替代 VLM reward 供 RL 训练使用，并在 GT 轨迹上验证与 VLM 的一致性

---

## 一、离线评测（步骤级 teacher-forcing，不跑 GALFIT）

**状态：已实现，已跑通。**

### 1.1 输入与流程

- **模型输入**：GT 历史状态（残差图 + 参数摘要 + 历史轮次摘要）
- **模型输出**：文本（CoT 分析 + JSON spec）
- **GT**：专家动作的 JSON spec + coarse_label

### 1.2 评价维度

| 维度 | 定义 | 值域 |
|------|------|------|
| format_ok | 能否从输出中解析出有效 JSON spec | bool |
| type_match | 粗动作类型（add/modify/delete）是否与 GT 一致 | bool |
| comp_count_match | 输出的成分数量是否与 GT 一致 | bool |
| param_scores | 匹配成分间参数的 tolerance-normalized 得分 | 每参数 0~1 |
| acc_score | param_scores 的综合均值 | 0~1 |

### 1.3 评分细节

**参数 tolerance 阈值**（单参数得分 = `clip(1 - |pred - gt| / tolerance, 0, 1)`）

| 参数 | tolerance | 说明 |
|------|-----------|------|
| mag  | 0.5 mag   | 通量 ±0.5 等级 |
| Re   | max(2px, gt×0.5) | 相对 50% 或绝对 2px，取大者 |
| n    | 1.0       | Sersic index ±1 |
| q    | 0.15      | 轴比 ±0.15 |
| pa   | 20°       | 位置角 ±20°（180° wrap：`min(|diff|, 180-|diff|)`） |

**组件匹配**：贪心匹配——按 role 精确匹配 +2 分，model 类型相同 +1 分，贪心选最高分配对。

**动作类型分类**：
- GT 标签归一化：`add_psf/add_disk/add_sersic → add`，`delete_* → delete`，其余 `modify`
- Pred 标签推导：从 spec 的 `target` 文字中用关键词匹配推导（添加→add，删除→delete，其余→modify）
- **无 stop 类型**：训练数据中不包含 stop，pipeline 终止由 patience+max_steps 外部控制

### 1.4 汇总报告与局限

**分项指标**（每一项都是数据集级的均值/比例，分开报告）：
- format_rate, type_accuracy, comp_count_match_rate
- param_scores（每参数 0~1 均值）+ param_mae（每参数原始 MAE）
- per_type_metrics（按 add/modify/delete 分组）+ per_depth_metrics（按 depth 分组）
- label_confusion（混淆矩阵）

**为什么不给一个综合指标**：类似 UI-S1 的 AndroidControl-High SR 需要天然二元的正确性（"点击是否落在对的区域"），我们参数是连续的——把 format/type/param 揉成一个数必须选权重或阈值，而我们没有客观依据（VLM ground truth 在离线评测阶段不参与）。**分项报告更诚实**：每维度自己就是一个可解释的指标，读者能看到模型在哪一维强、哪一维弱。

**综合指标在哪里出现**：在执行评测（2.4）里，因为那时有 VLM 校准的 threshold 可用（见 3.8），SSR 才不是拍脑袋。

**局限**：
- **只看文本层面**，不知道模型输出的参数跑 GALFIT 后效果如何
- **参数简并性问题**：参数差得远但实际拟合效果可能一样好，离线评测无法区分
- 适合快速迭代、对比不同训练阶段的模型

## 二、执行评测（步骤级 teacher-forcing + GALFIT 执行）

**状态：当前优先实现。** 在离线评测基础上增加 GALFIT 执行，用残差质量评价操作结果。

### 2.1 核心思路与流程

给模型 GT 历史状态作为输入（teacher-forcing），模型预测 action，**实际执行 GALFIT**，然后比较：
- 模型执行后的残差 vs GT 下一步的残差
- 模型输出的参数 vs GT 参数

本质是离线评测的升级版——同样的输入对齐方式，多了执行环节，能同时看参数和残差两个维度。

流程：
1. 从 GT 轨迹的第 k 步状态出发（残差图 + 参数摘要 + 历史）
2. 构造 prompt → 模型推理 → 得到 pred_text → 解析出 action spec
3. 执行 GALFIT → 得到模型的残差图 + metrics（chi2_nu, BIC 等）
4. 三维度评价（见 2.2）

### 2.2 评价维度

**(a) 参数距离**（沿用离线评测）：type_match, param_scores, acc_score

**(b) 残差质量比较**（模型残差 vs GT 同一步残差）：

| 指标 | 定义 | 说明 |
|------|------|------|
| residual_l2 | ‖model_res - gt_res‖₂ / n_pixels | 像素级 L2 距离，越小越好 |
| residual_ssim | SSIM(model_res, gt_res) | 结构相似度，越接近 1 越好 |
| model_noise_score | 模型残差的噪声质量（std_ratio + PSD） | 绝对质量，看残差是否接近白噪声 |
| gt_noise_score | GT 残差的噪声质量 | 参考基线 |
| noise_gap | model_noise_score - gt_noise_score | 跟专家的差距，越接近 0 越好 |

**(c) 指标对比**：

| 指标 | 定义 | 说明 |
|------|------|------|
| **bic_diff** | model_bic - gt_bic | **主判据**：0 = 一样，>0 = 比专家差 |
| chi2_ratio | model_chi2_nu / gt_chi2_nu | 诊断用：≈1 不代表模型正确（可能过拟合），BIC 才是最终判据 |

**为什么 BIC 是主判据（chi2 不是）**：
- `chi2_nu ≈ 1` 只说明"残差在噪声水平"，可以是**正确模型**、**过拟合**（参数多把噪声也拟了）、或**错模型**碰巧误差正好噪声级——三种情况 chi2_nu 都可能接近 1
- **BIC = χ² + k·ln(N)** 惩罚复杂度，同时看拟合优度和参数经济性——**BIC 降低才说明"这步操作有物理意义"**
- 常见反直觉场景：删掉一个冗余成分，chi2_nu 略升但 BIC 大降 → 是**好步骤**（VLM 也会认可），单看 chi2 会误判

### 2.3 关键优势与依赖

优势：
1. **回避参数简并性**：参数差很远但残差一样好 → 能识别出来，说明是合法的简并解
2. **每步独立评价**：因为 teacher-forcing 输入对齐，不存在误差累积
3. **定位问题更清晰**：参数差 + 残差也差 = 真的做错了；参数差 + 残差好 = 简并解

依赖：
- 需要 GALFIT 二进制可用（仅 A6000）
- 需要 GT 轨迹的每步残差图、sigma 图、mask 图
- 每步串行执行（推理 + GALFIT）

### 2.4 汇总报告与综合指标

**分项指标**：沿用 1.4 的所有指标，另加执行环节的产物：
- Mean chi2_ratio, Mean bic_diff, Mean noise_gap
- residual_l2 / residual_ssim 分布
- Mean RL reward（连续 rule reward 均值）

**综合指标 SSR (Step Success Rate)**：跟离线评测不一样，执行评测有 **VLM 校准的阈值**（3.8），所以能给一个不拍脑袋的综合数：

```
step_success = format_ok AND type_match AND rule_reward > threshold
SSR = (1/N) · Σ I[step_i success]
```

- `threshold` **不是拍的**：来自 val 集上跟 VLM ground truth 的对齐优化（见 3.8），语义等于"这步 rule 判定与 VLM 一致会通过"
- SSR 直接对应 UI-S1 AndroidControl-High 的 SR：**teacher-forcing 下的单步成功率**
- 分维度报告：Per-Type SSR（add/modify/delete）、Per-Depth SSR

## 三、RL Reward 设计与对齐验证

对齐 pipeline 中的 `calculate_reward_model_with_param`，用 rule-based 指标替代 VLM 视觉判断。核心目标：**在 GT 轨迹上对齐 VLM 判断，同时可扩展到大规模 RL 训练**。

### 3.1 设计思路

- **不依赖与 GT 的参数距离**（因为简并性——多组参数都能给好拟合）
- 评估"这步操作的结果好不好"：参数边界 + BIC/chi2 变化 + 残差噪声质量
- 各维度与 VLM reward 一一对应，方便对齐验证

**关于 chi2 与 BIC 的定位**（参照 2.2 的讨论）：
- **BIC 是主判据**：单看 chi2 会误判（chi2_nu ≈ 1 可能是过拟合，chi2 略升可能是合理的删冗余）
- **chi2 作为负向门槛用**：只在 chi2 明显恶化（>26%）时一票否决，避免"BIC 靠塌参数换低复杂度"的极端刷分
- **noise_score 独立评估**：残差是否接近白噪声，跟 chi2/BIC 关注点不同

**VLM reward 维度 → Rule-based 替代**：

| VLM reward 维度 | Rule-based 替代 | 说明 |
|---|---|---|
| 残差是否改善（视觉判断） | `robust_std_mad(res/sigma)` + PSD 平坦度 | 借鉴同事 batch_eval 代码 |
| 参数是否合理（不跑飞） | 边界检查（n∈[0.1,8], Re∈[0.2,300], q∈[0.05,1]等），**含拟合后值** | 硬门控 |
| chi2/BIC 趋势 | chi2_nu 对数增益 + BIC 变化（log 归一化） | 对齐 `calculate_reward_model_with_param` |
| 残差与参数图文一致性 | 去掉 | 纯 VLM 能力，rule 无法替代 |

### 3.2 组合公式

```
if 参数边界违规（spec 初值 或 拟合后值 越界）:
    R = 0    # 对齐 VLM Part 3 param_plausible
elif chi2/nu 明显恶化（r_chi2 < -0.1，即恶化 >26%）:
    R = 0    # 一票否决，对齐 VLM Part 4 Rule 3
else:
    R = W_CHI2 × r_chi2 + W_BIC × effective_r_bic + W_NOISE × r_noise
```

**分量定义**：
- `r_chi2 = log10(old_chi2_nu / new_chi2_nu)` — 改善为正
- `r_bic = sign(old_bic - new_bic) × log10(1 + |old_bic - new_bic|)` — 对数归一化
- `effective_r_bic`：`r_chi2 ≥ 0 → min(r_bic, BIC_CAP=2.0) × bic_damping`；否则 `min(r_bic, 0)`
- `bic_damping`：`parent_chi2_nu` 分区打折的启发式（见 3.5），只在 [0.9, 1.15] × 0.3 才明显生效
- `r_noise` = 残差噪声质量连续得分 0~1

**当前权重**（v11）：`W_CHI2=10.0`, `W_BIC=0.5`, `W_NOISE=5.0`。权重通过 `eval/validate_reward_alignment.py` 在 GT 轨迹 val 集上校准。

> **历史备注**：v7-v9 曾加过 `chi2_nu-zone BIC damping` 的外推；v10 曾加过 redundancy / negligible_flux 硬检测。两者都被数据证伪并回退（v9 damping 外推到过拟合区导致 FN +44；v10 硬检测把 VLM 举例数字当规则，导致 FN +173）。**最终版 v11 = v8**（保留 bounds 两层检查 + 3.5 的启发式 damping [0.9, 1.15] × 0.3）。教训：对齐 VLM 要看它的**判断行为**，不是文本里的举例数字。

### 3.3 参数边界检查（两层）

| 参数 | 合法范围 | 违规处理 |
|------|----------|----------|
| n (Sersic index) | [0.1, 8.0] | 硬门控 → R=0 |
| Re (有效半径) | [0.2, 300] px | 硬门控 → R=0 |
| q (轴比) | [0.05, 1.0] | 硬门控 → R=0 |
| mag (星等) | > 0 | 硬门控 → R=0 |

两层：
1. **spec 初值检查**：模型输出的 action_spec 里所有 component 都要在范围内
2. **拟合后检查**（v6 起）：从子节点 `summary_path` 解析 GALFIT 拟合后的 sersic/expdisk/psf 参数，同样应用上表边界

VLM 原文（Part 3）：`n < 0.1 or n > 8`, `Re < 0.2 pixel`, `q < 0.05 or q > 1.0`, `Re unreasonably large`。

**为什么拟合后检查是最大跳跃**（v5→v6）：VLM 看的是拟合后参数，rule 之前只查 spec 初值 → 拟合后 n=12/q=0.02/Re=0.02 全漏掉。加了拟合后检查后 AUC 0.85→0.89，FP 340→210。

**Rule 无法检测的 VLM Part 3 项目**（承认能力上限）：
- "redundant / nearly duplicated components" — 判断依赖上下文（bulge vs bar），rule 直接用距离+mag 差硬阈值 → v10 实测严重误伤，放弃
- "negligible-flux components" — VLM 举例 "mag diff > 5"，但实测 PSF 核比主星系暗 5-7 mag 是合理的，rule 硬阈值 → v10 实测严重误伤，放弃
- "large center offsets", "implausible size hierarchy", "more components without residual support" — 需要几何关系 / 类型分类 / 图像分析

### 3.4 噪声质量评估

借鉴同事 `batch_eval_residual_quality_hierarchical.py`。

**核心指标**：
1. `robust_std_mad(residual/sigma)` → 理想值 ~1
   - < 1 说明过拟合（残差波动小于噪声）
   - \> 1 说明欠拟合（残差波动大于噪声）
2. `psd_band_metrics()` → `low_to_high`, `mid_to_high` → 理想值 ~1（白噪声平坦谱）
   - `low_to_high` 偏大：残差有大尺度结构（成分缺失）
   - `mid_to_high` 偏大：中频结构

**阈值**（沿用同事默认值，后续可调）：
- std_low=0.6, std_high=1.5（硬门控）
- std_soft_low=0.80, std_soft_high=1.20（软警告）
- psd_low_tol=2.0, psd_mid_tol=1.5

### 3.5 chi2 / BIC 联动门控 + damping

**BIC-chi2 gate**（对齐 `calculate_reward_model_with_param` Part 4）：

| 情形 | chi2 gain | BIC gain | 处理 |
|------|-----------|----------|------|
| 正常改善 | > 0 | ≥ 0 | 正常正向 reward |
| 可能过拟合 | > 0 | 明显恶化 | BIC 负贡献抵消部分 chi2 |
| Metric-driven | ≈ 0 | 明显改善 | BIC 正贡献（对应 VLM Combined Rule B "metric-driven improvement"） |
| 恶化否决 | 明显恶化（r_chi2 < -0.1） | 任意 | R=0（VLM Part 4 Rule 3） |

**BIC damping**（v8 版，按 `parent_chi2_nu` 分区打折 BIC 正贡献）：

| parent_chi2_nu | 语义 | BIC 贡献系数 |
|---|---|---|
| [0.9, 1.15] | 拟合已在完美区 | × 0.3 |
| [0.8, 0.9) 或 (1.15, 1.4] | 接近完美 | × 0.5 |
| 其他 | chi2 还有明显改善空间 | × 1.0 |

**关于 damping 的诚实说明**：
- **不是 VLM 文本直译**——VLM 判过拟合是靠 param_plausible 类判据（"more components without residual support" 等），需要图像分析
- **是启发式近似**：rule 看不见视觉残差，用"拟合已在完美区 + BIC 大幅下降"作为"改善缺乏残差支持"的**代理信号**
- **数据支持保留**：v6→v7 加此 damping 时 FP -76 而 FN 只 +7（10:1 交换比，说明抓的是真信号）
- **不做外推**（v9 曾扩展到过拟合区 chi2<0.9，数据证伪：FN +44 只换 FP -26；v9 已回退）

### 3.6 对齐验证方法

pipeline 的 VLM reward 依赖视觉判断，无法用于大规模 RL 训练；上述 rule-based reward 用可计算量替代，本节在 GT 轨迹上验证一致性。

**代码入口**：`eval/validate_reward_alignment.py`

- **数据来源**：pipeline 已生成的 GT 轨迹，每步都有 VLM reward 记录
- **划分**：按 `physical_id` 划分 val/test（7:3），val 集调 threshold + 权重，test 集锁参评测
- **对齐指标**：
  - ROC AUC：连续 rule reward 的排序能力
  - Accuracy / Precision / Recall / F1：给定 threshold 下的二值对齐
  - 混淆矩阵：**FP（rule=1, VLM=0）= 教坏模型风险**；FN（rule=0, VLM=1）= 训练信号稀疏
  - 分动作类型（add/modify/delete）指标
  - 不一致样本导出（`val_disagreements.jsonl`）供人工核对

### 3.7 迭代实验记录

| 版本 | 关键改动 | AUC | Accuracy | FP | FN | Precision |
|------|---------|-----|----------|-----|----|-----------|
| v1 | 初版：raw BIC 差值直接加权 | 0.53 | -- | -- | -- | -- |
| v2 | BIC log-normalize：`sign(Δ)·log10(1+|Δ|)` | -- | -- | 445 | -- | -- |
| v3 | BIC-chi2 gate + Re 上限 + W_BIC 5→2 | 0.85 | 81.6% | 366 | 145 | 0.69 |
| v4 | 收紧 gate (r_chi2 ≥ 0.003) + BIC_CAP=2 | 0.83 | 77.2% | 581 | -- | -- |
| v5 | 回退 gate, W_BIC 2→0.5（均衡贡献方差） | 0.85 | 81.3% | 340 | 178 | 0.70 |
| v6 | 加拟合参数边界检查（sersic） | 0.89 | 87.1% | 210 | 148 | 0.80 |
| v7 | 多类型解析(sersic+expdisk+psf) + BIC damping [0.9,1.15]×0.3 | 0.91 | 89.6% | 134 | 155 | 0.86 |
| **v8**（最终） | 修 expdisk parser bug（4 值不是 5 值） | **0.91** | **90.1%** | **121** | **155** | **0.87** |
| v9 | damping 扩到过拟合区（**臆测，作废**） | 0.90 | 89.4% | 95 | 199 | 0.89 |
| v10 | 加 redundancy/negligible 检测（**误用 VLM 举例数字，作废**） | 0.86 | 85.5% | 73 | 328 | 0.90 |
| v11 | 退回 v8 + Precision-first F1 threshold | v8 相同 | ↓略 | ~80 | ~200 | **≥ 0.90** |

**关键洞察**：
- **v6 拟合边界检查是最大跳跃**：v5 的 340 个 FP 中 100% 是 `bounds_ok=True` 但 VLM 说 `unphysical`
- **v7 BIC damping**：拟合已收敛区 (chi2≈1) 时 BIC 改善打折。**是启发式，不是 VLM 直译**，但 10:1 FP/FN 交换比证明抓真信号
- **v8 expdisk parser bug**：GALFIT fit log expdisk 只有 4 值（无 n 占位），v7 的 `len>=5` 分支 silently drop 掉所有 expdisk
- **v9 失败教训**：把 v7 damping 外推到过拟合区（chi2<0.9）——VLM 没这条逻辑，是我脑补；数据证伪（FN +44）
- **v10 失败教训**：把 VLM Part 3 举例数字（mag diff > 5、dist < 2px）当硬阈值实现——VLM 用这些是 holistic 判断而非规则；186/200 新 FN 是误杀
- **v11 收工**：接受 v8 已经是 rule-based 天花板（剩余 FP 三类根因都需要图像分析），转而用 Precision-first F1 卡 threshold 到 precision ≥ 0.9

### 3.8 Threshold 策略选择

| 策略 | 目标 | 类别不平衡下的行为 |
|------|------|--------------------|
| Accuracy 最优 | (TP+TN)/n 最大 | 负样本多，倾向"少判正"降低 FP，但同时漏掉 recall。多数类主导。 |
| F1 最优 | 2·P·R/(P+R) 最大 | 只关心正类的 precision/recall 平衡。忽略 TN。 |
| Youden's J | TPR - FPR 最大 | 对称对待两类，ROC 曲线离对角线最远的点。 |
| **Precision-first F1** | max F1 s.t. precision ≥ 0.9 | 保 rule=1 里至少 90% 真正例，防止教坏模型。**当前采用** |

**选择理由**：
- RL 场景下 **FP（错误奖励坏动作）比 FN（错过好动作）后果严重得多**——FP 直接教坏模型走向 reward-gaming
- Accuracy 最优在负例多时偏"少判正"，本质是被 TN 主导，跟质量目标不同
- 纯 F1 忽略类别不平衡，可能反而降低 threshold，跟"压 FP"目标反向
- Precision-first F1 直接对应"rule=1 里尽量都是真好步骤"的诉求

### 3.9 RL 训练时的使用方式

**决策：二值化 reward**

```python
final_reward = 1.0 if rule_reward > threshold else 0.0
```

**理由**：
1. **训练/评测口径统一**：评测用二值对齐率作为指标，训练也用二值，避免"训练目标 ≠ 评测目标"
2. **标签源限制**：ground truth 是 VLM 的 binary `improvement`，我们没有更细粒度的连续质量标签。子维度（residual_improved / param_plausible / metric_consistent）是 VLM 内部思考步骤，不是独立的质量标签
3. **鲁棒性**：rule reward 有硬门控悬崖（bounds/veto → 0）+ BIC damping 分段常数，非单调风险实存，二值化能吸收这些风险
4. **对齐 GUI Agent 惯例**：UI-TARS S1 等 GUI Agent 论文均使用二值 reward

**日志约定**：训练日志同时记录**连续 rule_reward**，便于事后诊断 reward gaming（比如平均连续值涨到 outlier 值域但二值通过率没变 = 模型在刷分）。

## 四、SOP：半在线轨迹评测（模型自主历史）— 暂不实现

**术语澄清**：SOP = **Sequential Online Performance**（沿用 UI-S1 定义），不是 "Standard Operating Procedure"。核心思想：模型用**自己的**历史 rollout，遇到第一步"错步"即停，用完成进度衡量能力。

### 4.1 概念对齐

| UI-S1 | 我们的对应 |
|---|---|
| **AndroidControl-High**（teacher-forcing，每步用 GT 上下文，报 SR） | **二、执行评测**（2.4 SSR） |
| **SOP**（模型自己的历史，遇错就停，报 PG/TSR/Score） | **本章** |

跟执行评测的核心区别：
- 执行评测 = 每步都从 GT 状态出发（teacher-forcing），没有误差累积
- SOP = 从 GT root 出发，之后**每步基于模型上一步的输出**，误差会累积

### 4.2 定义（PG / TSR / Score）

对每条轨迹 i：
- `t_i` = 专家轨迹的总步数
- `s_i` = 模型能连续对齐的步数（直到第一次"错步"为止）

**核心指标**：

| 指标 | 定义 | 说明 |
|------|------|------|
| **PG** (Progress) | `(1/N) · Σ (s_i / t_i)` | 平均完成进度，[0,1]，对部分完成也给分 |
| **TSR** (Task Success Rate) | `(1/N) · Σ I[s_i = t_i]` | 严格全对率，只有整条轨迹都对齐才算成功 |
| **Score** | `(PG + TSR) / 2` | 综合分，兼顾"进展"与"完全成功" |

**"错步"定义**（我们的场景需要额外约定，因为参数是连续的）：

两个候选定义：
1. **Match-based**（严格）：动作类型 + 参数在 tolerance 内匹配 GT → 参数简并会误判，不推荐
2. **Quality-based**（推荐）：`rule_reward > threshold` → 容许简并解，只要"这步是好步"就继续

**倾向 quality-based**：更贴近"轨迹是否往好方向走"的直觉，避免参数简并把合法简并解误判为错步。

### 4.3 补充指标（终态质量）

即使 SOP 用"遇错即停"，还是可以对**跑完的部分**报终态相对质量：

| 指标 | 定义 | 说明 |
|------|------|------|
| **bic_diff** | model_final_bic - gt_final_bic | **主判据**：0 = 一样，>0 = 比专家差 |
| improvement_ratio | (init_bic - model_final_bic) / (init_bic - gt_final_bic) | 完成了专家 BIC 改善量的百分之几 |
| noise_gap | model_noise_score - gt_noise_score | 噪声质量差距 |
| chi2_ratio | model_chi2_nu / gt_chi2_nu | **诊断用**：≈1 不代表模型正确（见 2.2） |

**注意**：`improvement_ratio` 之前是基于 chi2 定义的，现在改成基于 BIC——BIC 才是"是否完成物理意义上分解"的度量，chi2 只能算"局部拟合优度"。

### 4.4 为什么暂不做

1. **前序步骤误差累积**：一旦前几步走偏，后续所有步骤都建立在错误状态上（SOP 用"遇错即停"缓解，但小 s_i 的分辨率就低了）
2. **终态比较缺乏基准**：GT 轨迹不一定有"接受的终态"——很多是 max_steps 或 patience 终止的，不是"做完了"
3. **当前模型水平**：离线评测显示 delete 准确率 11%、depth=2 决策 type_acc 46%，SOP 下大部分轨迹会在前几步停掉，PG 分辨率不够
4. **优先级**：先把执行评测（teacher-forcing）做好，确认模型的能力天花板；SOP 用来评估"能不能真的自主完成"，等模型底子够了再上

## 五、三个层次的关系

```
离线评测（快，不跑 GALFIT）
  ↓ 加 GALFIT 执行
执行评测（慢，跑 GALFIT，teacher-forcing）    ← 当前重点
  ↓ 去掉 teacher-forcing，模型自主历史
SOP 半在线评测（最慢，端到端）                ← 后续再做
```

| | 离线评测 | 执行评测 | SOP 半在线 |
|---|---|---|---|
| 输入对齐 | GT 状态 | GT 状态 | 模型自身上一步 |
| 跑 GALFIT | 否 | 是 | 是 |
| 综合指标 | 无（无 VLM 校准锚点，分项报告） | SSR（VLM 校准，2.4） | PG / TSR / Score（4.2） |
| 误差累积 | 无 | 无 | 有（遇错即停缓解） |
| 简并性处理 | 无法区分 | 残差好=合法简并解 | 同执行评测 |
| 速度 | 快（~10s/样本） | 中（~40s/样本） | 慢（~5min/轨迹） |
| 实现状态 | ✅ 已完成 | 🔧 待实现 | 📋 设计备忘 |
| 实现状态 | ✅ 已完成 | 🔧 待实现 | 📋 设计备忘 |
