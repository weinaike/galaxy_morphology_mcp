# 评测体系设计文档

## 概述

评测体系分为三个层次，加一个独立的 RL Reward 模块：

1. **离线评测（步骤级，不跑 GALFIT）** — 快速衡量模型输出与 GT 的文本匹配度
2. **执行评测（步骤级，跑 GALFIT）** — Teacher-forcing + GALFIT 执行，用残差质量 + 参数距离双维度评价（**当前优先实现**）
3. **自主轨迹评测（轨迹级，模型自主驱动）** — 端到端评估模型分解能力（**暂不实现**）
4. **RL Reward 设计与对齐验证** — 用 rule-based 指标替代 VLM reward 供 RL 训练使用，并在 GT 轨迹上验证与 VLM 的一致性

---

## 一、离线评测（步骤级 teacher-forcing，不跑 GALFIT）

**状态：已实现，已跑通。**

### 1.1 输入与流程

- **模型输入**：GT 历史状态（残差图 + 参数摘要 + 历史轮次摘要）
- **模型输出**：文本（CoT 分析 + JSON spec）
- **GT**：专家动作的 JSON spec + coarse_label

### 1.2 评价维度

| 维度 | 定义 | 值域 |
|------|------|------|
| format_ok | 能否从输出中解析出有效 JSON spec | bool |
| type_match | 粗动作类型（add/modify/delete）是否与 GT 一致 | bool |
| comp_count_match | 输出的成分数量是否与 GT 一致 | bool |
| param_scores | 匹配成分间参数的 tolerance-normalized 得分 | 每参数 0~1 |
| acc_score | param_scores 的综合均值 | 0~1 |

### 1.3 评分细节

**参数 tolerance 阈值**（单参数得分 = `clip(1 - |pred - gt| / tolerance, 0, 1)`）

| 参数 | tolerance | 说明 |
|------|-----------|------|
| mag  | 0.5 mag   | 通量 ±0.5 等级 |
| Re   | max(2px, gt×0.5) | 相对 50% 或绝对 2px，取大者 |
| n    | 1.0       | Sersic index ±1 |
| q    | 0.15      | 轴比 ±0.15 |
| pa   | 20°       | 位置角 ±20°（180° wrap：`min(|diff|, 180-|diff|)`） |

**组件匹配**：贪心匹配——按 role 精确匹配 +2 分，model 类型相同 +1 分，贪心选最高分配对。

**动作类型分类**：
- GT 标签归一化：`add_psf/add_disk/add_sersic → add`，`delete_* → delete`，其余 `modify`
- Pred 标签推导：从 spec 的 `target` 文字中用关键词匹配推导（添加→add，删除→delete，其余→modify）
- **无 stop 类型**：训练数据中不包含 stop，pipeline 终止由 patience+max_steps 外部控制

### 1.4 汇总报告与局限

汇总产物：
- format_rate, type_accuracy, comp_count_match_rate
- param_scores（每参数 0~1 均值）+ param_mae（每参数原始 MAE）
- per_type_metrics（按 add/modify/delete 分组）+ per_depth_metrics（按 depth 分组）
- label_confusion（混淆矩阵）

局限：
- **只看文本层面**，不知道模型输出的参数跑 GALFIT 后效果如何
- **参数简并性问题**：参数差得远但实际拟合效果可能一样好，离线评测无法区分
- 适合快速迭代、对比不同训练阶段的模型

## 二、执行评测（步骤级 teacher-forcing + GALFIT 执行）

**状态：当前优先实现。** 在离线评测基础上增加 GALFIT 执行，用残差质量评价操作结果。

### 2.1 核心思路与流程

给模型 GT 历史状态作为输入（teacher-forcing），模型预测 action，**实际执行 GALFIT**，然后比较：
- 模型执行后的残差 vs GT 下一步的残差
- 模型输出的参数 vs GT 参数

本质是离线评测的升级版——同样的输入对齐方式，多了执行环节，能同时看参数和残差两个维度。

流程：
1. 从 GT 轨迹的第 k 步状态出发（残差图 + 参数摘要 + 历史）
2. 构造 prompt → 模型推理 → 得到 pred_text → 解析出 action spec
3. 执行 GALFIT → 得到模型的残差图 + metrics（chi2_nu, BIC 等）
4. 三维度评价（见 2.2）

### 2.2 评价维度

**(a) 参数距离**（沿用离线评测）：type_match, param_scores, acc_score

**(b) 残差质量比较**（模型残差 vs GT 同一步残差）：

| 指标 | 定义 | 说明 |
|------|------|------|
| residual_l2 | ‖model_res - gt_res‖₂ / n_pixels | 像素级 L2 距离，越小越好 |
| residual_ssim | SSIM(model_res, gt_res) | 结构相似度，越接近 1 越好 |
| model_noise_score | 模型残差的噪声质量（std_ratio + PSD） | 绝对质量，看残差是否接近白噪声 |
| gt_noise_score | GT 残差的噪声质量 | 参考基线 |
| noise_gap | model_noise_score - gt_noise_score | 跟专家的差距，越接近 0 越好 |

**(c) 指标对比**：

| 指标 | 定义 | 说明 |
|------|------|------|
| chi2_ratio | model_chi2_nu / gt_chi2_nu | 1.0 = 跟专家一样好，>1 = 比专家差 |
| bic_diff | model_bic - gt_bic | 0 = 一样，>0 = 比专家差 |

### 2.3 关键优势与依赖

优势：
1. **回避参数简并性**：参数差很远但残差一样好 → 能识别出来，说明是合法的简并解
2. **每步独立评价**：因为 teacher-forcing 输入对齐，不存在误差累积
3. **定位问题更清晰**：参数差 + 残差也差 = 真的做错了；参数差 + 残差好 = 简并解

依赖：
- 需要 GALFIT 二进制可用（仅 A6000）
- 需要 GT 轨迹的每步残差图、sigma 图、mask 图
- 每步串行执行（推理 + GALFIT）

## 三、RL Reward 设计与对齐验证

对齐 pipeline 中的 `calculate_reward_model_with_param`，用 rule-based 指标替代 VLM 视觉判断。核心目标：**在 GT 轨迹上对齐 VLM 判断，同时可扩展到大规模 RL 训练**。

### 3.1 设计思路

- **不依赖与 GT 的参数距离**（因为简并性——多组参数都能给好拟合）
- 评估"这步操作的结果好不好"：参数边界 + chi2/BIC 改善 + 残差噪声质量
- 各维度与 VLM reward 一一对应，方便对齐验证

**VLM reward 维度 → Rule-based 替代**：

| VLM reward 维度 | Rule-based 替代 | 说明 |
|---|---|---|
| 残差是否改善（视觉判断） | `robust_std_mad(res/sigma)` + PSD 平坦度 | 借鉴同事 batch_eval 代码 |
| 参数是否合理（不跑飞） | 边界检查（n∈[0.1,8], Re∈[0.2,300], q∈[0.05,1]等），**含拟合后值** | 硬门控 |
| chi2/BIC 趋势 | chi2_nu 对数增益 + BIC 变化（log 归一化） | 对齐 `calculate_reward_model_with_param` |
| 残差与参数图文一致性 | 去掉 | 纯 VLM 能力，rule 无法替代 |

### 3.2 组合公式

```
if 边界违规（spec 初值 或 拟合后值 越界）:
    R = 0
elif chi2/nu 明显恶化（r_chi2 < -0.1，即恶化 >26%）:
    R = 0  # 一票否决，对齐 VLM "be conservative"
else:
    R = W_CHI2 × r_chi2 + W_BIC × effective_r_bic × bic_damping + W_NOISE × r_noise
```

**分量定义**：
- `r_chi2 = log10(old_chi2_nu / new_chi2_nu)` — 改善为正
- `r_bic = sign(old_bic - new_bic) × log10(1 + |old_bic - new_bic|)` — 对数归一化
- `effective_r_bic`：`r_chi2 ≥ 0 → min(r_bic, BIC_CAP=2.0)`；否则 `min(r_bic, 0)`
- `r_noise` = 残差噪声质量连续得分 0~1
- `bic_damping`：`parent_chi2_nu` 分区打折（见 3.5）

**当前权重**（v7）：`W_CHI2=10.0`, `W_BIC=0.5`, `W_NOISE=5.0`。权重通过 `eval/validate_reward_alignment.py` 在 GT 轨迹 val 集上校准，目标是最大化对 VLM 的对齐 AUC。

### 3.3 参数边界检查（两层）

| 参数 | 合法范围 | 违规处理 |
|------|----------|----------|
| n (Sersic index) | [0.1, 8.0] | 硬门控 → R=0 |
| Re (有效半径) | [0.2, 300] px | 硬门控 → R=0 |
| q (轴比) | [0.05, 1.0] | 硬门控 → R=0 |
| mag (星等) | > 0 | 硬门控 → R=0 |

**两层检查**（v6 起）：
1. **spec 初值检查**：模型输出的 action_spec 里所有 component 都要在范围内
2. **拟合后检查**：从子节点 `summary_path` 解析 GALFIT 拟合后的 sersic/expdisk/psf 参数，同样应用上表边界

**为什么要拟合后检查**：VLM 的 `param_plausible` 判断是基于拟合后的参数（比如 n=12.13 撞上限、Rs=0 塌缩），只查 spec 初值会漏掉这类"参数跑飞"的坏动作。v5→v6 引入拟合后检查是最大跳跃：AUC 0.85→0.89，FP 340→210。

### 3.4 噪声质量评估

借鉴同事 `batch_eval_residual_quality_hierarchical.py`。

**核心指标**：
1. `robust_std_mad(residual/sigma)` → 理想值 ~1
   - < 1 说明过拟合（残差波动小于噪声）
   - \> 1 说明欠拟合（残差波动大于噪声）
2. `psd_band_metrics()` → `low_to_high`, `mid_to_high` → 理想值 ~1（白噪声平坦谱）
   - `low_to_high` 偏大：残差有大尺度结构（成分缺失）
   - `mid_to_high` 偏大：中频结构

**阈值**（沿用同事默认值，后续可调）：
- std_low=0.6, std_high=1.5（硬门控）
- std_soft_low=0.80, std_soft_high=1.20（软警告）
- psd_low_tol=2.0, psd_mid_tol=1.5

### 3.5 chi2 / BIC 联动与 damping

**BIC-chi2 gate**（对齐 `calculate_reward_model_with_param`）：

| 情形 | chi2 gain | BIC gain | 处理 |
|------|-----------|----------|------|
| 正常改善 | > 0 | ≥ 0 | 正常正向 reward |
| 可能过拟合 | > 0 | 明显恶化 | BIC 负贡献抵消部分 chi2 |
| Metric-driven | ≈ 0 | 明显改善 | BIC 正贡献（对应 VLM metric-driven improvement） |
| 恶化否决 | 明显恶化 | 任意 | R=0（VLM Part 4 Rule 3） |

**BIC damping**（v7 起，按 `parent_chi2_nu` 分区打折 BIC 正贡献）：

| parent_chi2_nu | 含义 | BIC 贡献系数 |
|---|---|---|
| [0.9, 1.15] | 拟合已收敛，"完美区" | × 0.3 |
| [0.8, 0.9) 或 (1.15, 1.4] | "接近完美" | × 0.5 |
| 其他 | chi2 还有明显改善空间 | × 1.0 |

**为什么加 damping**：v6 剩余 FP 中 96% 的 `child_chi2_nu ≤ 1.05`——拟合已经接近完美（chi2/nu ≈ 1 是理论最优），再靠 BIC 改善的多半是过拟合或 BIC 刷分。

### 3.6 对齐验证方法

pipeline 的 VLM reward 依赖视觉判断，无法用于大规模 RL 训练；上述 rule-based reward 用可计算量替代，本节在 GT 轨迹上验证一致性。

**代码入口**：`eval/validate_reward_alignment.py`

- **数据来源**：pipeline 已生成的 GT 轨迹，每步都有 VLM reward 记录
- **划分**：按 `physical_id` 划分 val/test（7:3），val 集调 threshold + 权重，test 集锁参评测
- **对齐指标**：
  - ROC AUC：连续 rule reward 的排序能力
  - Accuracy / Precision / Recall / F1：给定 threshold 下的二值对齐
  - 混淆矩阵：**FP（rule=1, VLM=0）= 教坏模型风险**；FN（rule=0, VLM=1）= 训练信号稀疏
  - 分动作类型（add/modify/delete）指标
  - 不一致样本导出（`val_disagreements.jsonl`）供人工核对

### 3.7 迭代实验记录

| 版本 | 关键改动 | AUC | Accuracy | FP | Precision |
|------|---------|-----|----------|-----|-----------|
| v1 | 初版：raw BIC 差值直接加权 | 0.53 | -- | -- | -- |
| v2 | BIC log-normalize：`sign(Δ)·log10(1+|Δ|)` | -- | -- | 445 | -- |
| v3 | BIC-chi2 gate + Re 上限 + W_BIC 5→2 | 0.85 | 81.6% | 366 | 0.69 |
| v4 | 收紧 gate (r_chi2 ≥ 0.003) + BIC_CAP=2 | 0.83 | 77.2% | 581 | -- |
| v5 | 回退 gate, W_BIC 2→0.5（均衡贡献方差） | 0.85 | 81.3% | 340 | 0.70 |
| v6 | **加拟合参数边界检查**（sersic） | 0.89 | 87.1% | 210 | 0.80 |
| v7 | 多类型解析(sersic+expdisk+psf) + BIC damping | **0.91** | **89.6%** | **134** | **0.86** |

**关键洞察**：
- **v6 拟合边界检查是最大跳跃**：v5 的 340 个 FP 中 100% 是 `bounds_ok=True` 但 VLM 说 `unphysical`。VLM 看的是拟合后参数，rule 之前只检查 spec 初值 → 拟合后 n=12/q=0.02/Re=0.02 全漏掉
- **v7 BIC damping 逻辑**：v6 剩余 FP 中 96% 的 `child_chi2_nu ≤ 1.05`（拟合已收敛），进一步的 BIC 改善是过拟合或 BIC 刷分

### 3.8 Threshold 策略选择

| 策略 | 目标 | 类别不平衡下的行为 |
|------|------|--------------------|
| Accuracy 最优 | (TP+TN)/n 最大 | 负样本多，倾向"少判正"降低 FP，但同时漏掉 recall。多数类主导。 |
| F1 最优 | 2·P·R/(P+R) 最大 | 只关心正类的 precision/recall 平衡。忽略 TN。 |
| Youden's J | TPR - FPR 最大 | 对称对待两类，ROC 曲线离对角线最远的点。 |
| **Precision-first F1** | max F1 s.t. precision ≥ 0.9 | 保 rule=1 里至少 90% 真正例，防止教坏模型。**当前采用** |

**选择理由**：
- RL 场景下 **FP（错误奖励坏动作）比 FN（错过好动作）后果严重得多**——FP 直接教坏模型走向 reward-gaming
- Accuracy 最优在负例多时偏"少判正"，本质是被 TN 主导，跟质量目标不同
- 纯 F1 忽略类别不平衡，可能反而降低 threshold，跟"压 FP"目标反向
- Precision-first F1 直接对应"rule=1 里尽量都是真好步骤"的诉求

### 3.9 RL 训练时的使用方式

**决策：二值化 reward**

```python
final_reward = 1.0 if rule_reward > threshold else 0.0
```

**理由**：
1. **训练/评测口径统一**：评测用二值对齐率作为指标，训练也用二值，避免"训练目标 ≠ 评测目标"
2. **标签源限制**：ground truth 是 VLM 的 binary `improvement`，我们没有更细粒度的连续质量标签。子维度（residual_improved / param_plausible / metric_consistent）是 VLM 内部思考步骤，不是独立的质量标签
3. **鲁棒性**：rule reward 有硬门控悬崖（bounds/veto → 0）+ BIC damping 分段常数，非单调风险实存，二值化能吸收这些风险
4. **对齐 GUI Agent 惯例**：UI-TARS S1 等 GUI Agent 论文均使用二值 reward

**日志约定**：训练日志同时记录**连续 rule_reward**，便于事后诊断 reward gaming（比如平均连续值涨到 outlier 值域但二值通过率没变 = 模型在刷分）。

## 四、自主轨迹评测（模型自主驱动）— 暂不实现

### 4.1 为什么暂不做

1. **前序步骤误差累积**：一旦前几步走偏，后续所有步骤的评价都建立在错误状态上，很难判断是哪一步出了问题
2. **终态比较缺乏基准**：GT 轨迹不一定有"接受的终态"——很多轨迹是 max_steps 或 patience 终止的，不是"做完了"。因此无法定义"成功收敛"的绝对标准
3. **路径不对齐**：模型走自己的路径，跟 GT 路径无法逐步对比，中间步骤的评价只能看增量改善（跟上一步比），信息量有限
4. **当前模型水平**：从离线评测看 delete 准确率只有 11%，depth=2 决策 type_acc 只有 46%，放到自主轨迹里大概率前几步就走偏，结果没有区分度
5. **优先级**：先把执行评测（方案 B）做好，确认模型在 teacher-forcing 下的能力天花板，再看自主驱动的效果

### 4.2 设计备忘（供后续实现参考）

**流程**：
1. 从 GT 轨迹的 root 节点开始（初始 feedme + 初始残差图）
2. 每步：模型推理 → 解析 action → 执行 GALFIT → 计算 reward → 更新状态 → 下一步
3. 终止条件：噪声质量达标 / max_steps / patience（连续 N 步 reward 无改善）

**每步评价**：
- **RL reward**：跟上一步比（边界 + chi2/BIC gain + 噪声质量变化）
- **VLM reward**：pipeline 已有的 `calculate_reward_model_with_param()`（视觉比对前后残差）
- 两个都跑，验证 RL reward 与 VLM reward 的对齐程度

**终态评价**（跟 GT 终态做相对比较，因为模型可能都达不到理想值）：

| 指标 | 定义 | 说明 |
|------|------|------|
| chi2_ratio | model_final_chi2 / gt_final_chi2 | 1.0 = 跟专家一样好 |
| improvement_ratio | (init_chi2 - model_final) / (init_chi2 - gt_final) | 模型完成了专家改善量的百分之几 |
| noise_gap | model_noise_score - gt_noise_score | 噪声质量差距 |

注意：不是每条 GT 轨迹都有明确的"接受终态"，需要筛选或定义终态标准。

**汇总指标**：
- 每步平均 RL reward / VLM reward
- RL vs VLM 一致率
- 最终 chi2_ratio / improvement_ratio 的分布
- 收敛步数分布

## 五、SOP 级评测指标（参考 UI-S1）

参考 UI-TARS S1 的 SOP 评测范式：将星系分解轨迹视为一个 SOP（Standard Operating Procedure），每步操作是 SOP 中的一个 step。在整个数据集上汇总，报告以下指标。

### 5.1 离线评测 SOP 指标

| 指标 | 定义 | 说明 |
|------|------|------|
| Step Format Rate | 能解析出有效 JSON 的步骤占比 | 基本能力 |
| Step Type Accuracy | 动作类型（add/modify/delete）正确率 | 决策能力 |
| Step Param Score | 参数 tolerance-normalized 均值 | 参数精度 |
| Per-Type Accuracy | 按 add/modify/delete 分组的准确率 | 各类型表现 |
| Per-Depth Accuracy | 按轨迹深度分组的准确率 | 多步决策衰减 |

### 5.2 执行评测 SOP 指标

| 指标 | 定义 | 说明 |
|------|------|------|
| Step Success Rate (SSR) | 单步 RL reward > threshold 的步骤占比 | 核心指标 |
| Step Chi2 Match Rate | chi2_ratio ∈ [0.8, 1.2] 的步骤占比 | 拟合质量与 GT 可比 |
| Step Param Score | 同离线评测 | 参数距离 |
| Per-Type SSR | 按 add/modify/delete 分组的 SSR | 各类型表现 |
| Per-Depth SSR | 按轨迹深度分组的 SSR | 多步衰减 |
| Mean RL Reward | 全数据集步骤的平均 RL reward | 连续值汇总 |
| Mean Chi2 Ratio | 全数据集的平均 chi2_ratio | 偏离程度 |

### 5.3 自主轨迹评测 SOP 指标（暂不实现）

| 指标 | 定义 | 说明 |
|------|------|------|
| Trajectory Completion Rate | 噪声质量达标的轨迹占比 | 端到端成功率 |
| Mean Improvement Ratio | `(init_chi2 - final_chi2) / (init_chi2 - gt_final_chi2)` 均值 | 完成了专家改善量的多少 |
| Mean Trajectory Reward | 每条轨迹的累积 reward 均值 | 整体分解质量 |
| Mean Steps to Converge | 收敛步数均值 | 效率 |

## 六、三个层次的关系

```
离线评测（快，不跑 GALFIT）
  ↓ 加 GALFIT 执行
执行评测（慢，跑 GALFIT，teacher-forcing）  ← 当前重点
  ↓ 去掉 teacher-forcing，模型自主驱动
自主轨迹评测（最慢，端到端）               ← 后续再做
```

| | 离线评测 | 执行评测 | 自主轨迹评测 |
|---|---|---|---|
| 输入对齐 | GT 状态 | GT 状态 | 模型自身上一步 |
| 跑 GALFIT | 否 | 是 | 是 |
| 评价角度 | 参数距离 | 参数距离 + 残差质量 | 增量 reward + 终态质量 |
| 误差累积 | 无 | 无 | 有 |
| 简并性处理 | 无法区分 | 残差好=合法简并解 | 同执行评测 |
| 速度 | 快（~10s/样本） | 中（~40s/样本） | 慢（~5min/轨迹） |
| 实现状态 | ✅ 已完成 | 🔧 待实现 | 📋 设计备忘 |
