
要求严格遵循工作流开展星系拟合分析工作。

## workflow

1. 查看星系目录与原图分析
* **查看星系目录：** 确认所需的文件是存在的，包括 FITS 图像、掩膜文件、背景估计文件、feedme 文件等。
* **查看原始数据与图像：** 使用 view_original_image 工具分析原图的*_roi_2panel.png文件，确认星系的基本形态特征（如是否存在明显的核球、盘、棒结构等）。这将为后续的拟合提供重要的初始猜测值。

2. 结构搜索与动态校验 (Structure Search & Dynamic Validation)
*目标：确定星系包含哪些主要物理成分（核球、盘、棒、点源等），不追求参数极致收敛。*
- **执行拟合：**: 使用 `run_galfit`，执行galfit（要求输入配置文件绝对路径，若输入上一轮输出的galfit.xx, 可以复用上一轮的拟合结果）
- **日志诊断：** 使用 `analyze_fitlog` 工具分析拟合日志，检查是否存在参数简并、参数不确定度异常、参数触及边界等问题。
- **成分调整：** 使用`residual_analysis_by_vlm` 对残差图详细解读，调整成分类型和初始参数值，重新执行拟合。
- **闭环拟合:** 重复上述步骤，残差图无明显结构，且参数收敛合理后，进入下一阶段.

3. 物理意义分析与奥卡姆剃刀原则
    - 物理意义分析：遵循“物理意义分析与策略”章节内容逐条分析；保证每个成分的物理合理性。
    - 参考下文“奥卡姆剃刀原则”， 分析每个成分的物理合理性，避免过拟合；如果某些成分的参数极端或者不合理，需要调整参数或者成分，回到第二步重新拟合分析。
    - 如果残差无明显结果， 物理意义合理，且参数收敛，则可进入第4步。

4. 结果分析与报告撰写 
* 锁定最佳结果（符合物理意义且参数收敛的最优轮次）。并给出其对应的形态学物理意义（如：成分 A 代表经典的盘结构，成分 B 代表致密的核心星团）。
* 使用 `write_file` 工具将分析结论写入当前星系目录：`analysis_report_xxx.md`。
* **报告内容包含：**
    * **预处理信息：** 掩膜说明、背景设定依据。
    * **迭代过程流水账：** 每轮新增/删除成分的依据（多模态视觉判断记录）、参数发散与回退记录。
    * **最佳结果详情：** 最终采用的参数表、物理意义解读。
    * **附件索引：** 最佳轮次的目录路径、最终的 `feedme` 文件、最终的拟合对比图（原图/模型/残差）路径。
