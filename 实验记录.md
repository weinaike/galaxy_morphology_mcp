# 训练数据生成方案 — 对比实验记录

## 1. 实验目标

验证不同提议策略（rule_based / expert_guided / vlm_generated）及关键配置参数对训练数据生成质量的影响，为批量生产选定最优配置。

### 核心对比维度

| 对比 | 实验组 | 验证问题 |
|------|--------|---------|
| 提议策略价值 | E1 vs E3 | VLM提议相比规则提议能提升多少成功率？ |
| 专家规则对比 | E2 vs E3 | VLM提议能否匹配或超越基于Gadotti GT的专家规则？ |
| 专家提示注入 | E3 vs E4 | 在VLM prompt中注入Gadotti专家参数是否提升效果？ |
| Token成本优化 | E3 vs E5 | VLM并发调用从4路减到2路，成功率损失多少？ |
| 深层轨迹探索 | E3 vs E6 | 关闭贪心+放宽早停能否产出更多深层训练数据？ |

**E6 — vlm_generated + MH退火探索（深层轨迹）**
```python
PROPOSAL_STRATEGY = "vlm_generated"
USE_EXPERT_HINT_FOR_VLM = False
VLM_PROPOSAL_NUM_CALLS = 4
FORCE_GREEDY = False    # 关闭贪心，启用 Metropolis-Hastings 退火接受
MAX_PATIENCE = 2        # 保持默认，单变量对比
```

> **E6 设计动机**：E3 在 Step 4+ 成功率断崖归零，原因是严格贪心（`force_greedy=True`）不允许中间步暂时变差。
> 真实星系拟合中，加新component（如bar）常常导致chi2先升后降，需要多轮调参才能收敛。
> E6 仅改动一个变量（FORCE_GREEDY=False），与E3严格单变量对比：
>
> 1. **关闭贪心** (`FORCE_GREEDY=False`)：VLM判定 improvement=0 的变体仍有 ~13.5% 概率被接受
>    （Metropolis-Hastings: `P(accept) = exp(delta_r/T) = exp(-1/0.5) ≈ 0.135`），
>    允许pipeline走过"先变差再变好"的优化谷底，产出Step 4-6的训练样本。
>
> **E3 vs E6 关注的核心指标**：
> - 深层（Step 4-6）是否有 accepted 节点
> - 训练数据深度分布是否更均匀
> - SFT/DPO 样本数量变化（预期：E6 > E3）
> - 成本变化（预期：E6 ≈ E3，VLM调用数相同，仅搜索深度更深导致略多GALFIT运行）

## 5. 训练数据格式

### 5.1 SFT样本（监督微调正样本）

每条SFT样本 = 一个被accepted的树节点，训练模型学习"看到这样的残差图→应该输出这样的模型规格"。

```json
{
  "sample_id": "Plate0282_Fiber478_g__node_2_p0_v0",
  "galaxy_id": "SDSS_gband_Plate0282_MJD51658_Fiber478_g",
  "depth": 2,
  "coarse_label": "add_psf",
  "messages": [
    {
      "role": "system",
      "content": "你是一个星系形态拟合专家。根据当前拟合状态（参数表+残差图），输出下一轮完整模型规格..."
    },
    {
      "role": "user",
      "content": "## 当前拟合状态\n| 成分 | mag | Re | n | q | PA |\n|---|---|---|---|---|---|\n| Sersic_1 | 16.82 | 4.51 | 3.68 | 0.72 | 31.2 |\n\nchi2/nu=0.744 ...",
      "images": ["node_0_root_comparison.png"]
    },
    {
      "role": "assistant",
      "content": "```json\n{\"components\": [{\"role\":\"bulge\",\"model\":\"sersic\",\"mag\":17.5,\"re\":3.0,\"n\":4.0,\"q\":0.72,\"pa\":31.2,\"fix\":{\"n\":0}},{\"role\":\"nucleus\",\"model\":\"psf\",\"mag\":19.2,\"fix\":{}}],\"sky\":{\"value\":null,\"fix\":0},\"target\":\"添加PSF核点源拟合中心残差\",\"confidence\":0.90}\n```"
    }
  ],
  "metadata": {
    "parent_node_id": "node_0_root",
    "parent_chi2_nu": 0.744,
    "child_chi2_nu": 0.701,
    "reward_detail": {"improvement": 1, "confidence": 0.90, "param_plausible": true}
  }
}
```

**训练时**：input = system + user（含残差对比图），output = assistant（完整模型规格JSON）。

### 5.2 DPO样本（偏好对齐配对）

每条DPO pair = 同一父节点下的两个兄弟，一个被采纳（chosen）一个被拒绝（rejected），训练模型学习"区分好坏提议"。

```json
{
  "pair_id": "Plate0282__node_1_p0_v0_vs_v1",
  "galaxy_id": "SDSS_gband_Plate0282_MJD51658_Fiber478_g",
  "depth": 1,
  "prompt": [
    {"role": "system", "content": "你是一个星系形态拟合专家..."},
    {"role": "user", "content": "## 当前拟合状态\n...", "images": ["node_0_root_comparison.png"]}
  ],
  "chosen": "{\"components\":[{\"role\":\"bulge\",...},{\"role\":\"nucleus\",\"model\":\"psf\",\"mag\":19.2}],...}",
  "rejected": "{\"components\":[{\"role\":\"bulge\",...},{\"role\":\"disk\",\"model\":\"sersic\",\"mag\":16.5,\"re\":49.3}],...}",
  "metadata": {
    "winner_node_id": "node_1_p0_v0",
    "winner_coarse_label": "add_psf",
    "winner_chi2_nu": 0.701,
    "loser_node_id": "node_1_p0_v1",
    "loser_coarse_label": "add_disk",
    "loser_chi2_nu": 0.812,
    "loser_failure_reason": "param_runaway"
  }
}
```

**训练时**：input = prompt（相同的残差图+参数表），chosen = 好回答，rejected = 差回答。

### 5.3 抽取规则

**SFT**：遍历trajectory.json所有节点 → 跳过root → 跳过is_accepted=False → 跳过crash → 每个合格节点生成一条。

**DPO**：按parent_id分组 → winner池(is_accepted=True) × loser池(is_accepted=False且非crash) → 笛卡尔积配对 → 过滤confidence<0.7的loser。

## 6. 常用命令行

### 6.1 运行训练数据生成实验

```bash
# 修改 main_generate.py 中的 EXPERIMENT_ID 和对应配置后：
nohup python main_generate.py > exp_E3_vlm.log 2>&1 &

# 查看实时日志
tail -f exp_E3_vlm.log
```

### 6.2 生成轨迹可视化 HTML

```bash
# 单个星系
python visualize_trajectory.py output/E3__vlm_proposal_.../<galaxy_id>/<galaxy_id>_trajectory.json

# 批量生成（所有星系）
for f in output/E3__vlm_proposal_*/*_trajectory.json; do python visualize_trajectory.py "$f"; done

# 输出: 在trajectory.json同目录下生成 trajectory_viewer.html，浏览器打开即可查看
```

### 6.3 提取SFT/DPO训练数据

```bash
# 对某个实验的output提取训练数据
python -m data_gen.extract_training_data --input output/E3__vlm_proposal_gemini-3.1-pro-preview_vlm_reward_gemini-3.1-pro-preview_hist__20260629_145453/

# 指定输出目录
python -m data_gen.extract_training_data --input output/E6__vlm_.../ --output-dir ./extracted/E6/

# 输出文件: all_sft.jsonl, all_dpo.jsonl, extraction_report.json
```

### 6.4 成分预测准确率评测

```bash
# 评估某个实验的component prediction accuracy（与main分支70%同指标）
python -m data_gen.evaluate_component_accuracy \
  --input output/E3__vlm_proposal_.../ \

# 输出: component_accuracy.json (含精确匹配率、各成分P/R/F1、逐星系详情)
```
