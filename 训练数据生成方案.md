## Action 空间

训练数据生成支持三种提议策略，各策略的 Action 空间设计独立：

| 策略 | 动作格式 | 提议来源 | 适用场景 |
|------|---------|---------|---------|
| `rule_based` | A/B/C/D 离散动作 | 随机轮盘赌 + 物理保底 | 无先验标注数据 |
| `expert_guided` | A/B/C/D 离散动作 + 专家补丁 | 规则 + Gadotti_params.json | 有专家标注数据 |
| `vlm_generated` | 统一决策格式 | VLM 残差分析（并发多轮） | 大模型驱动 |

### 策略一：规则/专家先验型 Action（rule_based / expert_guided）

**设计思路：默认回调与增量覆盖**

为了防止在无标注的非专家数据上运行、或标注文件损坏时引发系统崩溃，Action 空间采用两级解耦机制：

1. **物理底线打底：** 物理重写层（`modify_feedme.py`）无条件继承天文专家设定的经典形态常数（如：Disk 的 n 默认强制锁定 1.0，Bulge 的 n 默认锁定 4.0）。
2. **先验增量覆盖：** 提议决策层（`proposal.py`）若检测到 `Gadotti_params.json` 存在，则将具体的专家物理真值转换为”数值补丁（Value Patch）”向下一级渗透，通过正则表达式强制改写默认槽位。

#### Action A: 结构探索与成分注入 (Add Component)

+ **动作意图：** 在当前模型的基础上增加一个新的 Sersic 或 PSF 成分。
+ **通信字典格式：**

```json
{
  “type”: “A”,
  “component_type”: “disk”,     // 必须。可选范围: [“bar”, “disk”, “bulge”, “psf”]
  “preset_n”: 1.0245,           // 可选。锚定专家真值并附带小幅正态探索噪声的 Sersic 指数
  “preset_re”: 12.3120,         // 可选。锚定专家真值的绝对有效半径（单位：像素）
  “mag_offset”: 1.2500          // 可选。初始光度调暗偏置
}
```

+ **物理层映射逻辑：**
    - 若字典中不含 `preset_*` 键：`modify_feedme.py` 依据 `component_type` 刷写保底参数（`disk`→n=1.0, `bulge`→n=4.0, `bar`→n=0.5）。
    - 若字典中含有 `preset_*` 键：物理层在完成保底参数写入后，立刻执行二次正则重写，将参数替换为补丁中的精确绝对物理数值。

#### Action B: 结构纠错与剪枝 (Remove Component)

+ **动作意图：** 消除当前模型中冗余或引发过拟合的物理组件。
+ **通信字典格式：**

```json
{
  “type”: “B”,
  “target_index”: 2             // 整数。1-based index，指明删除第几个组件
}
```

#### Action C: 参数空间微调与弱微扰 (Modify Parameters)

+ **动作意图：** 在骨架搭建完毕后，对已有组件的有效半径（$ R_e $）和 Sersic 指数（n）进行蒙特卡洛随机搜索。
+ **通信字典格式：**

```json
{
  “type”: “C”,
  “target_sersic_idx”: 0,       // 整数。0-based index，目标 Sersic 块
  “delta_re_factor”: 1.1200,    // 浮点数。乘性因子，新 Re = 旧 Re * factor
  “delta_n_val”: -0.3200        // 浮点数。加性增量，新 n = 旧 n + delta
}
```

+ **局限性：** Action C 仅控制 Re 和 n 两个参数，其余参数（x, y, mag, q, PA）从上一轮拟合结果直接回写，不施加增量。

#### Action D: 拟合终结判定 (Stop Fitting)

+ **动作意图：** 当残差图达到白噪声极限时，主动中断搜索树。
+ **字典格式：**`{“type”: “D”}`

#### 并发槽位分配

为了最大化利用单次并发的 `num_variants`（默认 16 个），系统根据搜索树的深度自适应调整动作比例：

| 树深度 (Step) | Action A (加) | Action B (减) | Action C (调) |
|---|---|---|---|
| Step 1~5 | 70% | 0% | 30% |
| Step 6+ | 5% | 5% | 90% |

### 策略二：VLM 完整模型规格型 Action（vlm_generated）

> **方案B 更新（当前实现）**：VLM 不再输出"增量决策"，而是**输出下一轮完整模型规格**——所有成分的绝对参数表（role/model/x/y/mag/re/n/q/pa）+ 逐参数 `fix` 标志（1=free,0=fixed）+ sky，对齐 MCP `component_analysis` 由 agent 写整份 feedme 的范式。我们据此用 `write_feedme_from_spec()` 写一份完整 feedme 再跑 GALFIT；训练用的结构化"动作标签"由 `diff_spec_vs_parent()` 对比父子成分反推（add_disk/add_bar/add_bulge/add_psf/modify/delete）。约束：相对父节点成分数 ±1。pass@k（N 路并发 + 按规格指纹去重）保留。
>

#### Action 空间审计与 coarse_label 分类（2026-06-16 更新）

**VLM 实际被允许输出的完整 action 空间**（按 build_proposal_prompt 的阶段四规范）：

| 维度 | 可能变化 |
|---|---|
| 成分数 | +1 / 0 / -1（每步 ≤1 个增减） |
| 每个成分的 `role` | bulge / disk / bar / nucleus / companion |
| 每个成分的 `model` | sersic / expdisk / psf |
| 每个成分的参数 | x, y, mag, re, n, q, pa（连续值） |
| 每个成分的 fix 标志 | 5 个参数各自 1=free / 0=fixed |
| sky | value（或 null 继承）；fix 恒 0 |

**当前 coarse_label（6 类）vs. 实际行为对照**：

| coarse_label | 触发条件 | 覆盖完整性 |
|---|---|---|
| `add_disk` / `add_bar` / `add_bulge` / `add_psf` | C == P+1，按新成分 role/model 派发 | ✅ 4 类已覆盖成分数+1 |
| `delete` | C == P-1 | ✅ 覆盖成分数-1（但 VLM 不主动提议，由后处理构造） |
| `modify` | C == P，参数或 fix 发生变化 | ⚠️ **过宽**：把所有"成分数不变"的变化都塞进同一个桶 |

**已发现的遗漏（待补 coarse_label）**：

| # | 待补标签 | 触发场景 | 训练意义 |
|---|---|---|---|
| 1 | `swap_model` (例: `swap_sersic_to_psf`) | C == P，但某位置 model 类型变了（sersic→psf 简化点源、sersic→expdisk 改盘等） | **高**：意味着父节点的 model 选错，父节点应标 `superseded`，不入 SFT 正样本 |
| 2 | `fix_change` | C == P，model 不变、参数值无显著变化、仅 fix 标志翻转（释放 Bulge n、锁 Bar n=0.5） | **高**：MCP 推荐的"先 fix 再 free"两段式策略，单独成桶才能统计命中率 |
| 3 | `relabel_role` | C == P，model 不变，role 变化（如 sersic-bulge → sersic-bar，自动锁 n=0.5） | 中：类 swap_model 的"修正"语义，影响 fix 自动应用 |
| 4 | `add_with_realloc` | 跟 `add_*` 同步发生，并修改了已有成分参数（通量重分配） | 已有 `add_with_realloc_count` 计数；作为 `add_*` 的子标签存在 diff 里即可，不必新增桶 |

**Action 到训练数据的映射**：

| coarse_label | SFT 正样本 | DPO winner（accepted） | DPO loser（rejected） | 备注 |
|---|---|---|---|---|
| `add_*` | ✅ | ✅ | ✅ | 主力训练信号；包含通量重分配的复合操作 |
| `modify` | ✅ | ✅ | ✅ | 参数微调/通量重分配 |
| `fix_change` | ✅ | ✅ | ✅ | 高质量训练信号（两段式 fix→free） |
| `swap_model` | ⚠️ **父节点要标 superseded，移出 SFT 正样本池** | ✅ | ✅ | 蕴含"上一步错了"，破坏链路连续性的正确性 |
| `relabel_role` | ⚠️ **父节点可能 superseded** | ✅ | ✅ | 跟 swap_model 类似 |
| `delete` | ✅（后处理构造） | ✅ | n/a | 不来自 VLM，由人工在正确链路里注入冗余成分构造 |

**核心规则**：
- 检测到 `swap_model` 或 `relabel_role` 时，**父节点标 `superseded_by_swap=True`**，训练数据抽取时跳过该节点作为 SFT 正样本，但可保留作为 DPO loser
- `add_with_realloc` 不作为独立标签，仍归入 `add_*`，但 `diff` 字段记录通量重分配细节
- 多成分同步变化（如同一步既加成分又改 fix）按主导变化派发；统计层面通过 `analytics.coarse_label_compound_count` 单独跟踪

**设计思路：对齐 MCP `component_analysis` Phase 4 输出**

MCP 的残差分析工具在 Phase 4 输出的是**一个统一决策**——包含结构变更（增/删/维持，每步最多 ±1 个成分）和所有成分的完整参数调整表。策略二直接采用这一范式，取代 A/B/C/D 离散动作格式，解决以下局限：

- A/B/C/D 无法表达**跨成分参数耦合**（如”添加 Bar 同时调整 Disk Re”）
- Action C 仅控制 Re 和 n，无法独立调整 mag、q、PA 等参数
- 单次调用返回多个提议 → 多样性不足且需复制填充

#### 统一决策格式

每次 VLM 调用输出 **1 个统一决策**：

```json
{
  “structural”: “add_bar”,         // 结构变更。可选: “add_bar”|”add_disk”|”add_bulge”|”add_psf”|”none”（”delete_N” 暂停用，见下）
  “new_component”: {               // 仅 structural 为 add_* 时有效
    “type”: “bar”,
    “model”: “sersic”,
    “mag_offset”: 1.5,
    “re”: 6.0,
    “n”: 0.5,
    “q”: 0.3,
    “pa”: 120.0,
    “bind_xy_to”: 0                // 绑定 x/y 到已有成分（0-based index）
  },
  “param_updates”: [               // 增量数组，可同时调整多个成分
    {
      “target_idx”: 0,             // 0-based Sersic 成分索引
      “delta_re_factor”: 1.2,     // 乘性因子
      “delta_mag”: -0.3,          // 加性增量
      “delta_q_factor”: 0.9,      // 乘性因子
      “delta_pa_val”: 5.0         // 加性增量（度）
    }
  ],
  “confidence”: 0.85,
  “reasoning”: “残差图中心区域呈一字型亮区，需添加 Bar 成分”
}
```

**与 A/B/C/D 的语义映射：**

| 统一决策 | 等价旧动作 | 说明 |
|---------|-----------|------|
| `structural: “add_*”` | Action A | 添加成分，**通常同时携带 param_updates 对已有成分做通量重分配**（A 的语义已扩展，不再是"纯加成分"） |
| `structural: “delete_N”` | Action B | 删除第 N 个成分。**当前暂停用**：VLM 不主动提议删除，删成分训练数据由后处理构造（见下） |
| `structural: “none”` + 有 param_updates | Action C | 纯参数调整，支持多成分多参数（预计低频） |
| confidence < 0.3 或空决策 | Action D | 等价于收敛终止 |

> **关于 delete（Action B）的处理**：`delete_N` 在 `parse_unified_decision()` / `apply_unified_action_to_feedme()` 中仍有完整解析与执行能力，但 `VALID_STRUCTURAL` 与阶段四 prompt **不再向 VLM 提供 delete 选项**。删成分的训练数据改由**后处理构造**——在过滤后的正确链路中人工插入一个错误的疑似成分，则"删除该错误成分"自然成为下一步的正确 action。这样无需在线生成删除提议，也无需改动 acceptance 逻辑。

> **Action A 语义说明（重要）**：由于添加成分时几乎必然伴随通量重分配（将原单成分的光分配给新旧两个成分），统一决策的 `add_*` 默认是"增加成分 + 调整已有成分参数"的复合操作，对齐 MCP `component_analysis` Phase 4 的专家行为，而非"两个独立 action 的序列"。统计上：主分类按 structural 归为 A，并额外记录 `add_with_realloc_count`（携带 param_updates 的 add 决策数）与按成分类型细分的 `structural_distribution`。

**相比 A/B/C/D 的增强：**
- param_updates 支持 mag、q、PA 的独立调整（Action C 仅支持 Re 和 n）
- 一个决策可同时包含结构变更和参数调整（Action A/C 互斥）
- fix/free 不由 VLM 控制，沿用代码层的原始标志保留行为

#### 多样性机制：并发多轮调用 + 去重

```
VLM Call 1 (temp=0.3) ──→ Decision 1 ─┐
VLM Call 2 (temp=0.5) ──→ Decision 2 ──┤── 去重 ──→ K 个唯一决策 ──→ 各跑 1 次 GALFIT
VLM Call 3 (temp=0.7) ──→ Decision 3 ──┤
VLM Call N (temp=0.9) ──→ Decision N ─┘
```

- **调用次数 N** 为超参数（`VLM_PROPOSAL_NUM_CALLS`，默认 4），类似 rule_based 的 `num_variants`
- 每次调用使用**不同 temperature**（从 0.3 到 0.9 线性插值），提高输出多样性
- N 次调用**并发执行**，单次失败自动重试 1 次
- 对 N 个结果按 `structural + param_updates 目标成分集合` 去重
- **不用 rule_based 填充**——去重后有几个唯一决策就跑几个 GALFIT 变体


### 已知风险与局限（vlm_generated 策略）

#### 1. fix/free 标志不可控

当前统一决策格式**不包含 fix/free 控制能力**，VLM 无法指示 GALFIT 固定或释放特定参数。代码层保留原始 feedme 中的 fix/free 标志（`ofix = stripped.split()[2]`）。这导致以下 MCP 推荐的调参策略无法执行：

- **Sky-fix**：MCP 要求将 sky 参数 fix 到 background 值（`fix=0`），防止 sky 漂移导致整体拟合偏移。当前系统依赖初始 feedme 中 sky 的 fix/free 状态，VLM 无法干预。
#### 1. fix/free 标志不可控　【已在方案B解决】

~~当前统一决策格式不包含 fix/free 控制能力~~。**方案B（完整模型规格）后，VLM 输出的每个成分带逐参数 `fix` 标志（1=free / 0=fixed），sky 恒 fix 到背景值，Bulge 可先 fix n=4 再 free，Bar n=0.5 强制 fixed**——sky-fix 与 Bulge Re 坍缩的多轮策略均已可表达。

#### 2. Disk 模型：方案B 已改用 expdisk

~~系统统一使用 sersic 表示所有成分~~。**方案B 起，对齐 MCP：Disk 用 expdisk（VLM 给 Re，写盘自动转 Rs=Re/1.678），Bulge/Bar 用 sersic，Nucleus/AGN 用 psf。** `parse_components_from_feedme()` 支持解析 sersic/expdisk/psf 三种模型。

#### 3. VLM prompt 相比 MCP 知识库的精简

当前 `vlm_proposal.py` 的 SYSTEM_PROMPT 是 MCP `residual_analysis_message.md` 的精简版，缺少以下细节：

| 缺失内容 | 潜在影响 |
|---|---|
| 旋臂识别规则（正负相间螺旋图案） | 可能误判旋臂残差为需要新成分 |
| 尘埃带/并合遗迹判定 | mask 问题被误判为成分问题 |
| 伴星系详细认定条件 | 可能过度添加伴星系成分 |
| 奥卡姆剃刀 BIC 门槛 | 不适用——由 reward 机制替代 |

> 注：通量分配 3:7/4:6 规范**已补入** build_proposal_prompt 阶段三参数分析，不再属于缺失项；其残余风险见风险 #5。

这些精简对训练数据生成的基本流程不阻塞，可在实际运行中根据 VLM 决策质量按需补充。

#### 4. parse_summary() 仅解析 sersic 和 sky

`galfit_actions.py` 中的 `parse_summary()` 只从 fit.log 提取 sersic 行和 sky 行，不解析 expdisk / ferrer / psf 成分。这与当前系统全部使用 sersic 的设计一致，但如果未来引入 expdisk 支持，需要同步扩展该函数的正则解析。

#### 5. 单轮合并 Prompt vs MCP 多轮分阶段，且无历史 action 上下文

MCP `component_analysis` 原本通过 **3 次 API 调用（4 阶段）** 逐步完成分析，并可携带历史拟合轨迹/上一步 action 作为上下文。当前系统将 4 阶段**压缩进单轮单 prompt**，且**不提供历史 action 序列**（每步独立决策，VLM 不知道上一步做了什么、参数如何演化）。

**潜在影响**：VLM 仅凭单张残差图判断参数量级，误差较大。实测中出现新增 Disk 的 Re=40px（专家真值仅 12.67px）、把已有成分 n 从 2 推到 5.56 等显著偏离，导致添加成分后残差几乎无改善。

**缓解方案**：新增 `USE_EXPERT_HINT_FOR_VLM` 开关，将 Gadotti_params.json 中的成分参数（Re/mag/n/q）作为初值锚点注入 prompt（仅 vlm_generated 策略生效），约束 VLM 的参数量级判断，同时保留其独立做结构决策的自由度。

**专家提示后的残余风险**：
- **绝对 mag → mag_offset 的转换由 VLM 完成**。专家提示给的是各成分绝对 mag（如 mag_disk=17.03），但 `new_component` 用的是相对第一个 sersic 的 `mag_offset`（`new_mag = 第一个sersic当前mag + mag_offset`）。VLM 需自行做减法得到 offset，这一步可能算偏，导致新成分通量初值仍不准。
- **无专家数据时风险完全保留**。当开关关闭、或星系无 Gadotti_params.json 时，仅剩 prompt 阶段三的泛化 3:7/4:6 比例规则，无绝对锚点，新成分初值质量不稳定。

#### 6. 成分中心联动（同心 x,y 绑定）　【方案B已实现】

~~执行层未实现真正的 GALFIT 参数联动~~。**方案B 起，`write_feedme_from_spec()` 在写出含 ≥2 个同心成分的 feedme 时，自动生成 `.cons` 约束文件**（对齐 MCP `component_specification_galfit.md` 的 `offset` 约束写法）：把所有非伴星系成分的 x、y 用 `offset` 绑定，并将 feedme 的 `G)` 行指向该 `.cons`（GALFIT 以 cwd=feedme 目录运行，用 basename 即可，`run_galfit` 会自动解析并归档约束文件）。伴星系（companion）有独立中心，不参与绑定。`pa` 也已通过完整规格正确写入。

#### 7. 边缘盘（edgedisk）暂不支持

**边缘盘（edge-on disk）**：指几乎沿盘面方向（视线在盘平面内）观测的盘星系，外观是一条细长亮带；其投影光度分布与正向盘不同，GALFIT 用专门的 `edgedisk` 模型拟合（参数为 mu0 中心面亮度、h_s 标高、R_s 标长、PA，与 sersic/expdisk 的 mag/Re/n/q 结构完全不同）。判据：一个 Disk 的 b/a < 0.5 可认定为 edge-on。

当前 `write_feedme_from_spec` / `parse_components_from_feedme` / prompt **只支持 sersic / expdisk / psf 三种模型，不含 edgedisk**。经核查所给的 working_note 样例（Plate0270/0271/0274/0276、obj104）均非边缘盘，出现概率低，故暂不实现；若后续数据中边缘盘比例上升，再补 edgedisk 模型块与对应 prompt 知识。

#### 8. BIC：直接引用 MCP 计算结果，不自行计算

经核查，**MCP 实际使用的是 1D BIC**（`extract_summary_galfit.py:496-501`），公式为 `bic1d = chisq1d + nfree·ln(n1d)`，其中 `chisq1d` 是 **1D 表面亮度轮廓（径向）** 上的卡方、`n1d` 是径向数据点数（约几十个，非 2D 像素数）。2D BIC（`chi2 + nfree·ln(ndof)`，ndof=像素数）在 src 中**已被注释、不使用**。这是天文老师的修正——用像素数作惩罚会被像素量级淹没，改用 1D 轮廓点数才有模型选择意义。

因此本系统**不自行计算 BIC**：`_extract_metrics()` 直接从 `run_galfit` 产出的 summary markdown 的 `| BIC |` 行解析 MCP 算好的 `bic1d`（同时解析 `χ²₁D/ν`）。reward 若需 BIC 把关（奥卡姆，ΔBIC>10 仅用于 Nucleus/伴星系），应使用此 `bic1d`，不得用 2D 版本。

### 训练数据生成算法工作流 (GALFIT 版)
#### Step 1: 初始化与基准建立 (Initialization)
+ **动作**：读取 SExtractor 给出的单 Sersic 模型作为 $ \theta_0 $。
+ **执行**：调用 GALFIT 跑出初始结果。
+ **采集**：调用采集器获取初始状态 $ S_0 $ 和初始复合基准分 $ R_0 $。
+ **记录**：初始化轨迹列表 `trajectory = []`。

#### Step 2: 提议分布与动作抽样 (Proposal Generation)
代码在此处需要写一个随机轮盘赌（或者简单的启发式策略），从定义的 A、B、C、D 四个 Action 中抽取多个。

+ **概率设定建议**：
    - **Action C (Adjust Params - 15%)**：代码随机挑选一个已有的 `component_id`，对其 $ R_e $ 或 n 施加一个高斯扰动 $ \Delta \theta \sim N(0, \sigma^2) $。
    - **Action A (Add Component - 70%)**：五步内最常发生，代码随机从可用类型库中挑一个（例如 `disk-like Sérsic`），将其中心坐标初始化在主星系中心。
    - **Action B (Remove Component - 10%)**：当 $ C_t > 1 $ 时触发，随机删掉一个通量贡献最小的组件。
    - **Action D (Stop - 5%)**：随机触发终止试探。

> 已支持三种提议策略：rule_based（随机轮盘赌）、expert_guided（专家先验补丁）、vlm_generated（VLM 并发多轮决策）。
>
> 上述概率可按照不同步数设置不同的概率组合，比如5步内 Action A 概率最大，5步外 Action C 和 Action D 概率最大。
>
> vlm_generated 策略下，每步并发调用 VLM N 次（超参数 `VLM_PROPOSAL_NUM_CALLS`），去重后各跑 1 次 GALFIT。
>
> action 规范参考：[https://www.yuque.com/manni-tt033/neh6ga/driqqgxf3n9rxg2i](https://www.yuque.com/manni-tt033/neh6ga/driqqgxf3n9rxg2i) 2. Assistant Output
>

#### Step 3: 执行动作与物理模拟 (Execution)
+ **动作**：根据 Step 2 抽到的动作，调用写好的 `add_component()`, `del_component()`, 或 `adj_params()` 函数，修改 `.feedme` 文件。
+ **执行**：拉起 GALFIT 引擎跑一轮拟合。
+ **采集**：引擎跑完后，立刻调用“信息采集器”，收集最新的各项指标，构建新状态 $ S_{new} $。

#### Step 4: 复合评价机制 (Reward Evaluation)
> 后续支持大模型打分。目前已实现三种 Reward 模式：
> - Mode 1: 纯规则（chi2 对数增益 + 边界惩罚）
> - Mode 2: VLM 双图对比（残差图视觉判断）
> - Mode 3: VLM 双图 + 参数合理性审查（残差改善但参数跑飞时降级为 improvement=0）
>

根据 `state_context`，计算 5 个维度的 Reward：

1. $ R_{step} $** (结构奖励)**：如果本次是 Action A，判断步数 t 是否超限，给 $ \pm 1 $。
2. $ R_{bound} $** (边界惩罚)**：遍历刚跑出的参数，如果进入 5% 死亡区，$ R_{bound} = -10 $。
3. $ R_{\chi^2} $** (拟合优度)**：$ \Delta \chi^2 = \chi^2_{old} - \chi^2_{new} $，计算对数或线性增益。
4. $ R_{residual} $** (核心惩罚)**：计算内部掩膜区域的方差，乘以 $ -\beta $。
5. $ R_{prior} $** (偏离惩罚)**：计算当前参数与 SExtractor 初始值的 L2 距离，乘以 $ -\gamma $。

最终汇总得分：

$ R_{total} = w_1 R_{step} + w_2 R_{bound} + w_3 R_{\chi^2} + w_4 R_{residual} + w_5 R_{prior} $

> 这些不同维度的reward需要相同数量级
>

_(注：如果未来引入大模型打分，就在这一步把 _`_state_context_`_ 转成自然语言 Prompt，让大模型输出一个 Score 代替上述公式。)_

#### Step 5: 接受准则与数据打标 (Acceptance & Logging - DPO 核心)
等待当前步的 N 个动作并发执行完毕，获得 N 个新状态和 $ \Delta R $。

计算提议动作的收益差值：$ \Delta R = R_{total}(new) - R_{total}(old) $

引入温度参数 T（设积极小，代表贪婪），计算接受率：

$ \alpha = \min \left( 1, \exp \left( \frac{\Delta R}{T} \right) \right) $

+ **如果接受 (Random(0,1) **$ \le \alpha $**)**：
    - 更新基准：状态变为 $ S_{new} $，基准分变为 $ R_{total}(new) $。
    - **落盘**：将此次的 Action JSON 作为 $ y_{winner} $ 记录进轨迹库，标记为优秀操作。
+ **如果拒绝 (Random(0,1) **$ > \alpha $**)**：
    - 撤销操作：利用函数回滚刚才修改的 `.feedme` 文件。
    - **落盘**：将此次的 Action JSON 作为 $ y_{loser} $ 记录进轨迹库，标记为典型的踩坑操作。

#### Step 6: 循环与终止
回到 Step 2，继续生成下一个动作。直到步数超过设定上限（如 10 步），或者抽中了 Action D（Stop），则结束当前星系的轨迹生成。

#### Step7：训练数据获取及数据过滤
抽取最终叶子节点的父节点，形成多轮轨迹数据，通过VLM残差图进行打标，判断是否拟合好了。
